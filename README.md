## üéØ Mandatory Sections Implementation

The project includes all five mandatory sections with the required content and functionality:

### 1. Conference (Home Page)

* **General Info:** Location, dates, main theme, and past editions are presented.
* **Committees:** Organizing Committee and Technical Committee (simulated data) are listed.
* **Sponsors:** Major sponsors and partners are showcased.
* **Image Gallery:** A gallery section features simulated archival images from past events/presentations.

### 2. Program

* **Schedule:** A detailed daily program is provided, including regular sessions, special sessions, and social events.
* **Themes:** All main conference themes are presented in a dedicated section.
* **Keynote Speakers:** Profiles of the keynote speakers are featured in a grid layout.

### 3. Paper Submission

* **Information:** Details on important dates, submission procedures, and paper templates are available.
* **Contact Form:** Includes a dedicated form to contact the organization.
    * **Fields**: Collects name, gender, ID card number, address, email, and phone number.
    * **Reason Select**: Allows the user to select the reason for contact (Paper Submission, Registration, Other Questions).
    * **Conditional Input**: If "Other Questions" is selected, an additional text input appears to specify the topic.

### 4. Registration

* **Pricing:** Details on event prices and registration procedures are provided.
* **Logistics:** Information regarding location/facilities, transportation, and contacts is included.
* **Touristic Material:** Features a slideshow element showcasing local touristic material related to the location.

### 5. Membership

* **Login/Registration:** This section allows existing users to log in and new users to register.
* **Simulated Authentication:** The entire authentication/registration process is simulated using Vanilla JavaScript (client-side).
